Terminal-Led Development

2026-03-12

Speed-of-thought engineering only comes when you give up the ikky GUI addiction.

The setup

I run my entire workflow from the terminal. LazyVim, tmux, lazygit, Claude Code, Omarchy. No IDE. No Electron apps. No mouse.

It’s faster. But that undersells it. When you’re jumping between panes, staging hunks, diffing changes, running AI agents in a split, you feel like a god. Like playing an instrument. You learn new chords every week and you never stop getting faster.

Everything is text

Customisation is just text config and shell scripts. I needed to insert a file from my clipboard into a project. One keybinding. Done. Try doing that in VS Code without installing three extensions and reading a forum thread.

The tooling has caught up

LazyVim gives you good defaults without the config hell. lazygit makes reviews fast. diffview makes diffs easy to read. This stuff used to take days to set up. Now you install a distro and you’re going in minutes. When you get stuck, AI helps you fix it.

AI fits right in

Claude Code fits right in. Runs in a tmux pane alongside everything else. Reads files, writes code, runs tests. MCPs, custom skills, agents. It works with the workflow, not around it.

People ask why I don’t just use Cursor. I tried it. Felt like going backwards.
